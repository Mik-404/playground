# multiboot [100 баллов]
После загрузки ядра bootloader'ом, процессор оказывается в самом начале функции `_early_entry_point` (в [`arch/x86/early.S`](../arch/x86/early.S)). Он имеет определённое состоние, которое описано в [спецификации Multiboot2](https://www.gnu.org/software/grub/manual/multiboot2/multiboot.html), секция "I386 machine state". Например, `eax` всегда содержит специальное магическое значение `0x36d76289`, а `ebx` содержит указатель на структуру, которая сгенерирована загрузчиком. В этой структуре есть довольно много всякой полезной информации, однако, нас пока будет интересовать информация о доступной памяти.

Цель этого задания — прочитать документацию и распарсить карту памяти, которая предоставлена бутлоадером. Для этого вам нужно:
1. Реализовать загрузку Boot Info и поиск multiboot тэгов, реализовав функции `multiboot::Init` и `multiboot::LookupTag` в [`kernel/multiboot.h`](../kernel/multiboot.h) соответственно.
2. Реализовать read-only итератор, который позволяет проходить по всем регионам памяти и получать информацию о них. Интерфейс итератора описывается классом `MemoryMapIter` в файле [`kernel/multiboot.h`](../kernel/multiboot.h). Пример использования можно найти в [`kernel/main.cpp`](../kernel/main.cpp) или в [`tests/test_multiboot.cpp`](../tests/test_multiboot.cpp).

Тесты лабораторной работы компилируются с помощью флага `TEST_MULTIBOOT`. Запускать можно и без них – если вы правильно напишете итератор, обычный запуск HellOS тоже должен отработать нормально.
