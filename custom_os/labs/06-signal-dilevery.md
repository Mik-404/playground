
# signal-delivery [200 баллов]
Сигналы являются одним из ключевых механизмов в POSIX, позволяющим уведомлять процессы о различных событиях. В HellOS уже реализована доставка сигналов процессам с помощью функций `SignalSend` и `SignalDeliver`. Однако, поскольку `arch::SignalDeliver` возвращает `ENOSYS`, доставка любого незаблокированного сигнала приводит к завершению процесса.

В рамках этого задания вам необходимо обеспечить корректную доставку сигналов процессам с возможностью их обработки. В нашей минимальной версии libc уже предусмотрены заготовки для обработки сигналов на x86, которые находятся в `user/stdlib/arch/x86/signal.S` и `user/stdlib/signal.c`. Со стороны операционной системы требуется лишь вызвать зарегистрированный обработчик сигнала (информация о них хранится в `sched::Task::sigactions`) и реализовать системный вызов `SysSigreturn`, который позволяет процессу вернуться в прежний контекст выполнения. Обработчики сигналов должны использовать тот же стек, на котором работает основной поток выполнения программы.

Тесты для этой задачи работают в user space. Чтобы их запустить, достаточно добавить флаг `TEST_SIGNAL` при сборке через `caos-make`:
```
caos-make clean && caos-make -j... TEST_SIGNAL=1
```
