Реализация канонического алгоритма **LR-k** постоения таблиц и парсинга текста для некоторого подкласса контестно свободных грамматик. 

Параметр K (длина предпросмотра) фиксируется во время компиляции, что позволяет добиться высокой производительности генератора таблиц и парсера. 

Работа строится таким образом:
 1. Компилятору на вход подается грамматика в установленном формате, об этом ниже. Он строит lr-k таблицы для парсинга, а затем сохраняем их, и ещё некоторую необходимую информацию для парсера.
 2. Парсер на вход получает результат работы генератора и строку, которую наобходимо разобрать, если строка лежит в языке, который задается грамматикой, поданной на вход компилятору, то на выходе получается последовательность правил в обратном порядке в правостороннем выводе, применяя которые, получается исходное слово (нумерация совпадает с исходной).

**Формат грамматики**

 * Старторый нетерминал всегда `S`
 * Правила вида `S -> a S b D`, токены разделены пробелами. До стрелки - lhs, после - rhs.
 * eps-правила в таком и только таком виде `S -> ` или `S -> eps`

В тексте для разбора все токены также должны быть разделены пробелами.

**Зависимости**
 * C++20 совместимый компилятор (GCC 10+)
 * [`argparse v3.2`](https://github.com/p-ranav/argparse/releases/tag/v3.2) установливается при сборке
 * [`googletest v1.16.0`](https://github.com/google/googletest/releases/tag/v1.16.0) установливается при сборке
 * `gcovr 7.0` нужно отдельно установить и добавить в path

**Инструкция по запуску**
1.  Клонируйте репозиторий:
    ```sh
    git clone https://github.com/Mik-404/playground.git
    cd <имя-папки>
    ```
2.  Сконфигурируйте проект с помощью CMake:
    *   Обычная сборка (K=1 по умолчанию):
        ```sh
        cmake -S . -B build
        ```
    *   Сборка с конкретным K (например, K=5):
        ```sh
        cmake -S . -B build -D K=5
        ```
    *   Сборка с генерацией покрытия (Coverage):
        ```sh
        cmake -S . -B build -D K=1 -D ENABLE_COVERAGE=ON
        ```

3.  Соберите проект:
    ```sh
    cmake --build build
    ```

Теперь проект собран и можно запустить тесты, в папке `build`

    ctest
    cmake --build . --target coverage

После этого появится `build/coverage/coverage_report.html`. Здесь можно найти подробную информацию о покрытии кода.

Для обычного запуска, из папки build

    ./Compiler --input grammar.txt
    ./Parser --input_table table --text input.txt

Также можно использовать для получения подробной информации о запуске

    ./Compiler --help
    ./Parser --help
