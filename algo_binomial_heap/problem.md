### Биномиальная куча

**Полное решение**

| Ограничения | Все языки (make) | Java 21 (Temurin JDK) |
| :--- | :--- | :--- |
| **Ограничение времени** | 1 секунда | 3 секунды |
| **Ограничение памяти** | 256Mb | 512Mb |
| **Ввод** | стандартный ввод или input.txt |
| **Вывод** | стандартный вывод или output.txt |

В этой задаче вам предстоит реализовать биномиальную кучу. Не используйте библиотечные функции, реализуйте все методы самостоятельно.

Обработайте запросы следующих видов:

*   `0 a x`: добавить элемент с целым значением *x* в кучу с номером *a*. Вновь добавленный элемент имеет уникальный индекс, равный порядковому номеру соответствующей операции добавления. Нумерация начинается с единицы.
*   `1 a b`: переложить все элементы из кучи с номером *a* в кучу с номером *b*. После этой операции куча *a* становится пустой.
*   `2 i`: удалить элемент с индексом *i*.
*   `3 i x`: присвоить элементу с индексом *i* значение *x*. Гарантируется, что *i*-й элемент существует.
*   `4 a`: вывести значение минимального элемента в куче с номером *a*. Гарантируется, что куча непуста. Время ответа на запрос такого типа должно составлять **O(1)**!
*   `5 a`: удалить минимальный элемент из кучи с номером *a*. Если таковых несколько, выбирается элемент с минимальным индексом. Гарантируется, что куча непуста.

#### Формат ввода

Первая строка содержит два целых числа *n* и *q* (1 ≤ *n* ≤ 1000, 1 ≤ *q* ≤ $10^6$) — общее количество куч и количество запросов, соответственно. Изначально все кучи пусты.

В следующих *q* строках содержатся запросы в описанном выше формате. Добавляемые числа *x* лежат в диапазоне [1, $10^9$].

#### Формат вывода

Для каждой операции поиска минимального элемента выведите его значение.

#### Пример

**Ввод**
```
3 19
0 1 10
4 1
0 2 5
0 2 7
4 2
3 2 20
4 2
1 2 1
4 1
5 1
4 1
3 2 3
4 1
2 2
4 1
0 1 9
1 1 3
0 3 8
4 3
```

**Вывод**
```
10
5
7
7
10
3
10
8
```
